# Промпт на улучшение проекта

Ты работаешь над проектом `md2pdf`: CLI-конвертером Markdown в PDF A4 portrait с Mermaid и оформлением, близким к ГОСТ Р 7.32.

Цель: навести инженерную красоту без раздувания проекта.

Сделай следующее:

- Проведи ревью `src/md2pdf.py` и раздели код на понятные модули только там, где это реально уменьшает сложность.
- Сохрани текущий визуальный контракт PDF: A4 portrait, Times New Roman, кегль 14, интервал 1,5, абзацный отступ 1,25 см, простые таблицы, нумерация страниц снизу.
- Не добавляй тяжелый web-render pipeline, если текущий ReportLab-подход решает задачу надежнее.
- Улучши CLI: понятные ошибки, проверка зависимостей, опции для полей, кегля, максимальной высоты диаграмм и директории временных файлов.
- Добавь минимальные тесты на парсинг таблиц, списков, заголовков и выбор Mermaid-рендера.
- Обнови README так, чтобы новый пользователь мог установить зависимости и собрать `README.pdf` без дополнительных догадок.
- Сохрани Markdown простым: только относительные ссылки, без абсолютных локальных путей.

Перед завершением выполни:

```bash
npm install
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
python3 src/md2pdf.py README.md README.pdf
python3 src/md2pdf.py examples/example.md examples/example.pdf
```
